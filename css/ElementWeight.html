# css层叠性

  1.         div {
            color: red;
            font-size: 20px;
        }
         
        div {
            font-size: 25px;
        }

    最终效果 就近样式覆盖原有样式

  2.         div {
            color: red;
            font-size: 25px;
        }

  3. 原则

    1. 样式冲突，遵循的原则是就近原则。 那个样式离着结构近，就执行那个样式。
    2. 样式不冲突，不会层叠
  4. CSS层叠性：长江后浪推前浪，前浪死在沙滩上(就近原则)

    (1) 在选择器中不要使用ID。就算只用一个ID，也会大幅提升优先级。当需要覆盖这个选择器时，通常找不到另一个有意义的ID，于是就会复制原来的选择器，然后加上另一个类，让它区别于想要覆盖的选择器。 (2) 不要使用！important。它比ID更难覆盖，一旦用了它，想要覆盖原先的声明，就需要再加上一个！important，而且依然要处理优先级的问题。 关于重要性的一个重要提醒 当创建一个用于分发的JavaScript模块（比如NPM包）时，强烈建议尽量不要在JavaScript里使用行内样式。如果这样做了，就是在强迫使用该包的开发人员要么全盘接受包里的样式，要么给每个想修改的属性加上！important。 正确的做法是在包里包含一个样式表。如果组件需要频繁修改样式，通常最好用JavaScript给元素添加或者移除类。这样用户就可以在使用这份样式表的同时，在不引入优先级竞赛的前提下，按照自己的喜好选择编辑其中的样式

# css继承性

  1. 子元素继承父元素样式(子承父业)

  2. 继承样式是部分的，而且不是全部

    1. 文本属性：
color、
font、font-family、font-size、font-weight、font-variant、font-style、
line-height、letter-spacing、
text-align、text-indent、text-transform、
white-space以及word-spacing。

    2. 列表属性：list-style、list-style-type、list-style-position以及list-style-image。

    3. 表格的边框属性border-collapse和border-spacing。注意，这些属性控制的是表格的边框行为，而不是常用于指定非表格元素边框的属性

  3. 继承可以优化或简化代码，降低代码读写和复杂度及阅读性

# css权重

  1. 权重计算公式

    选择器 计算权重公式
    继承性，通用选择器(*),关系选择器 0,0,0,0
    元素选择器,伪元素选择器 0,0,0,1
    类选择器,伪类选择器,属性选择器 0,0,1,0
    ID选择器 0,1,0,0
    行内样式 style="" 1,0,0,0
    !important ∞ 无穷大

  2. 权重金字塔

    T0：!important

    T1：行内样式

    T2: ID

    T3: 类 , 伪类 , 属性

    T4: 元素 ， 伪元素

    T5: 通配符,后代,父子,相邻兄弟，普通兄弟,选择器列表

    T6：继承性，层叠性

  3. 权重大小

    !important  >行内样式>ID>(类,伪类,属性)>(元素,伪元素)>(通配符,后代,父子,相邻兄弟，普通兄弟,选择器列表)>(继承性，层叠性)

    1. !important>行内样式>ID

    2. 类>元素>关系

    3. 伪类>伪元素>关系

    4. 属性>元素>关系>继承

  4. 数位之间没有进制 比如说： 0,0,0,5 + 0,0,0,5 =0,0,0,10 而不是 0,0, 1, 0， 所以不会存在10个div能赶上一个类选择器的情况。
