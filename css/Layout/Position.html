# 浮动布局

# 浮动

一个元素应沿其容器的左侧或右侧放置，允许文本和内联盒子元素环绕它

float ： left | right | none | inline-start | inline-end
属性值 描述
left 元素在当前块盒子左边浮动
right 元素在当前块盒子右边浮动
none 不浮动
inline-start 元素在当前块盒子开始一边浮动
inline-end 元素在当前块盒子结束一边浮动

# 清除浮动

clear： none | left | right | both | inline-start | inline-end
属性值 描述
left 清除左边浮动
right 清除右边浮动
none 不清楚浮动
both 清除俩边浮动
inline-start 清除开始一边浮动
inline-end 清除结束一边浮动

# 位置

# 顶部

top：<length> | <percentage> | auto
  • 当position设置为absolute或fixed时，top属性指定了定位元素上外边距边界与其包含块上边界之间的偏移。
  • 当position设置为relative时，top属性指定了元素的上边界离开其正常位置的偏移。
  • 当position设置为sticky时，如果元素在 viewport 里面，top属性的效果和 position 为relative等同；如果元素在 viewport 外面，top属性的效果和 position 为fixed等同。
  • 当position设置为static时，top属性无效。
属性值 描述
<length> 长度值数据类型 设置偏移位置
<percentage> 百分比数据类型 设置偏移位置
auto 通过浏览器自动计算偏移位置。

# 堆叠顺序

z-index：auto | <integer>
属性值 描述
auto 堆叠顺序与父元素相等
<integer> 整型数据类型顺序 大于0页面靠后，等于0就近页面，大于0页面靠前

# 定位

position：static | relative | absolute | sticky | fixed
属性值 描述 是否脱离文档 是否占位置 偏移基准 是否能转行内块 页面滚动
static 静态定位（默认） 不脱离 占 不偏移 不能 随着页面移动
relative 相对定位 不脱离 占 自身偏移位置 不能 随着页面移动
absolute 绝对定位 脱离 不占 父级元素偏移位置 能 随着页面移动
fixed 固定定位 脱离 不占 浏览器偏移位置 能 固定页面位置
sticky 粘性定位 不脱离 不占 父级偏移位置 能 页面滚动超出区域，会固定位置

# 静态定位

元素使用正常的布局行为，即元素在文档常规流中当前的布局位置。 元素加上 top, right, bottom, left 和 z-index 属性无效

# 相对定位

元素以及它周围的所有元素，都还保持着原来的位置，再不改变页面布局的前提下调整元素位置。 （此元素未添加定位时所在位置留下空白） 元素加上 top、right、bottom 和 left 属性，元素就会从原来的位置移走，但是不会改变它周围任何元素的位置

# 绝对定位

元素会被移出正常文档流，并不为元素预留空间. 通过指定元素相对于最近的非 static 定位祖先元素的偏移，来确定元素位置。 绝对定位的元素可以设置外边距（margins），且不会与其他边距合并 元素加上 "left", "top", "right" 以及 "bottom" 属性进行移动本身位置。

# 固定定位

元素会被移出正常文档流，并不为元素预留空间. 通过指定元素相对于屏幕视口（viewport）的位置来指定元素位置。 元素的位置在屏幕滚动时不会改变。打印时，元素会出现在的每页的固定位置。fixed 属性会创建新的层叠上下文。当元素祖先的 transform, perspective 或 filter 属性非 none 时，容器由视口改为该祖先 元素加上"left", "top", "right" 以及 "bottom" 属性移动本身位置。

# 粘性定位

元素定位类似相对定位; 而当页面滚动超出目标区域时，它的定位转换固定定位，它会固定在目标位置
